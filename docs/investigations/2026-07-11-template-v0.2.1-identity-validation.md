**Template branch:** `template/main`

British English. Neutral role language. No personal emails, no-reply addresses, local paths, hostnames, secrets or AI authorship metadata.

---

## 1. Problem corrected

The v0.2.0 distributable template hardwired the maintainer's public GitHub handle (`gromitski`) and required that identity in Git authority rules. That made the template unsuitable for other individuals, organisations, private projects or repositories where Git identity is not yet configured.

Slice 0.5.1 generalises repository content to be identity-neutral while preserving privacy-safe Git behaviour through reusable policy and initialisation checks.

---

## 2. Identity model

| Layer | Policy |
| ----- | ------ |
| **Repository content** | Identity-neutral unless the project deliberately publishes ownership information. Use the maintainer, the project owner or the user. No hardwired handles or emails in distributable files. |
| **Git metadata** | Uses each clone's configured privacy-safe Git identity. This maintainer repository continues to commit with `gromitski` and configured GitHub no-reply metadata only — not written into documentation. |
| **Initialisation** | May inspect `git config user.name` and `git config user.email`. Must not copy values into project memory or README. Continues even if identity is missing or appears unsafe. Warns concisely before any future commit. Does not print unsafe values in full. |

No setup questionnaire. No identity placeholders. No committed identity configuration file.
